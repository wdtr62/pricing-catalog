How to use this catalog

Read this like a simple guidebook. You do not need to press “Save”—the site remembers your work for you as you go. This sheet is only available after you unlock; visitors only see short hints when they hover parts of the page.

Your work is saved automatically

Every time you change a price, a title, or a picture, the page stores it right away inside your web browser on this computer. Closing the tab or the browser does not erase your catalog, as long as you use the same browser and do not clear site data.

There is no cloud backup built in. Your list does not automatically sync to your phone or another laptop. If you need a backup, ask the person who set up the site about exporting or copying your data.

Two modes: looking and editing

  • Locked (visitor mode): Anyone can scroll, open pictures, and read prices. They cannot change your catalog.
  • Unlocked (your mode): After you enter the password, you can add categories, add products, type prices, and attach images.

Unlock and lock the editor

  1. Click Unlock pricing at the top.
  2. Type the password you were given (it is stored in the site files—change it before sharing the link widely).
  3. When you are finished editing, click Lock so the catalog is read-only for visitors again.

Adding products

Single products live in their own section. Use Add single product for one-off items that are not part of a themed group.

Categories are groups (for example “Bowls” or “Winter 2025”). Click Add category, then name the group. Inside each category, use Add image to add another listing card.

Putting photos on each item

  1. Copy your image files into the images folder that sits next to this website’s files (same idea when you publish to GitHub Pages).
  2. On each card, type the path, for example images/my-photo.jpg, or click Browse… and pick a file.
  3. You can also drag a picture from your computer and drop it onto the big preview area— it fills in the path for you.

More than one photo for the same item

Click Add another view to add extra rows. Each row is one photo. Use the ‹ and › arrows on the card to flip through views. In the big preview window, use the same arrows or the left and right arrow keys on your keyboard.

Prices, sets, and “sold”

  • Price (each): What one piece costs when sold by itself.
  • Offer this category as a set: Optional bundle price for the whole group. When you mark a piece as Sold, its individual price is subtracted from the bundle total so “remaining set price” stays fair.
  • Not sold individually: For pieces that only go with the set—check this so buyers know.
  • Sold: Check when an item is no longer available; the site can hide or flag it and update set math.

Thumbnails and details

Thumbnails shrinks a section into small picture tiles so you can scan quickly. Details & prices expands it again so you can read and edit full cards. This works for whole categories and for single products.

Keep your data safe

  • Do not rely on private or incognito windows for long-term storage—they may forget everything when closed.
  • Clearing “cookies and site data” for this site will remove your catalog from this browser.
  • Use the same browser on the same computer to see the list you built.

Back up and move to your website (GitHub Pages or any host)

Your catalog (titles, prices, image paths, categories) is saved automatically in this browser as you work. Use Copy JSON when you want the same data in a catalog.json file in your project. The actual picture files live in your images folder on disk.

  1. While editing, your list is saved automatically in the browser. You only need Copy JSON / Import JSON when moving to another computer, backing up, or putting data into the catalog.json file in your project.
  2. Click Copy JSON, then in your code editor create or open catalog.json next to index.html, paste, and save. Commit and deploy — a public site cannot write into your project folder by itself.
  3. Include the images folder in the repo with the same file names your paths use.
  4. On the live site, unlock, Import JSON…, paste the same JSON (or open catalog.json in an editor, copy all, paste). The page reloads; editing works the same.
  5. If catalog.json is on the server, first-time visitors (empty browser data) can load it automatically. After that, each visitor’s browser keeps its own copy until they import again or you publish updates.
